# Token-Efficiency Fixture Corpus

Frozen local corpus for the Token Efficiency axis ([#1256](https://github.com/shaun0927/openchrome/issues/1256), Epic [#1254](https://github.com/shaun0927/openchrome/issues/1254)). 5 archetypes × 10 instances = **50 fixtures**, each with a ≥ 12-field ground-truth JSON.

See [`RUBRIC.md`](./RUBRIC.md) for the per-field category breakdown and the precise "present" matching rule.

## Why templated synthetic HTML, not scraped real pages

The corpus is generated by [`scripts/bench/generate-token-fixtures.mjs`](../../../../scripts/bench/generate-token-fixtures.mjs) — it is author-owned templated HTML, not snapshots of third-party pages. This is a deliberate trade-off:

| Property | Templated synthetic | Real-page snapshots |
|---|---|---|
| License / ToS safety | Trivial — we author it | Per-site audit, may force a swap mid-sprint |
| Reproducibility | Byte-identical from the generator | Frozen at capture date; cannot be re-derived if the source site changes |
| Structural realism | Carries representative chrome (nav, sidebar, footer, hydration shims, related-content noise) so raw HTML dwarfs the structured fields and the compression ratio reflects a real page workflow | Highest |
| Adversarial coverage | Generator can be extended to inject noise patterns specific to each archetype | Limited to whatever happens to exist in the captured page |

The fixtures are deliberately padded with realistic page chrome so the compression ratio reflects a real "12 fields out of a full page" workflow and not a stripped-down benchmark cherry-pick. Each fixture is typically 10–40 KB of HTML so a 12-field structured payload of ~80–160 tokens against an 8000+ token raw HTML produces a meaningful ratio.

## Archetypes

Each archetype carries 12 ground-truth fields spanning all four `RUBRIC.md` categories (structured data / primary content / navigation+interactive / metadata).

### `ecommerce/` — product detail pages
`title`, `price`, `brand`, `sku`, `rating`, `reviewCount`, `availability`, `primaryCta`, `shippingNote`, `category`, `color`, `warranty`

### `news/` — news article pages
`headline`, `author`, `publishedDate`, `section`, `summary`, `wordCount`, `primaryCta`, `canonicalUrl`, `category`, `readingTime`, `language`, `breadcrumb`

### `docs/` — technical documentation pages
`title`, `apiVersion`, `category`, `summary`, `lastUpdated`, `primaryCta`, `canonicalUrl`, `breadcrumb`, `codeLanguage`, `language`, `nextPage`, `prevPage`

### `spa/` — single-page-app feed pages
`appName`, `userHandle`, `primaryAction`, `postTitle`, `postBody`, `likeCount`, `replyCount`, `shareCount`, `timestamp`, `authorAvatarAlt`, `navigationPrimary`, `searchPlaceholder`

SPA fixtures carry hydration shims (40 inline `<script type="application/json">` blocks) so the compression ratio reflects the real "framework boilerplate vs the actually visible content" delta that real React/Vue apps exhibit.

### `search-results/` — search-engine result pages
`query`, `totalResults`, `result1Title`, `result1Url`, `result1Snippet`, `result2Title`, `result2Url`, `result2Snippet`, `result3Title`, `result3Url`, `paginationNext`, `filterPrimary`

## Adding fixtures

Edit the corresponding `*_VARIANTS` array in `scripts/bench/generate-token-fixtures.mjs`, then:

```bash
node scripts/bench/generate-token-fixtures.mjs
```

The generator is deterministic — a given variant array produces byte-identical HTML, so commit the regenerated `.html` + `.ground-truth.json` files together. Tests load the committed files, never the generator output, so a forgotten regen would be caught immediately by the corpus tests under `tests/benchmark/run-token-efficiency.test.ts`.

## Field name → category mapping

Per [`RUBRIC.md`](./RUBRIC.md), each fixture must span all four field categories.

| Category | Example fields |
|---|---|
| structured data | `price`, `sku`, `brand`, `rating`, `reviewCount`, `wordCount`, `codeLanguage`, `likeCount` |
| primary content | `title`, `headline`, `summary`, `postTitle`, `query` |
| navigation / interactive | `primaryCta`, `breadcrumb`, `nextPage`, `paginationNext`, `filterPrimary`, `navigationPrimary` |
| metadata | `canonicalUrl`, `category`, `language`, `apiVersion`, `lastUpdated`, `appName`, `authorAvatarAlt` |
